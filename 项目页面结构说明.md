# 项目页面结构说明

## 页面关系说明

### 1. **App.vue** - 项目入口
- **作用**：项目的根组件，只包含 `<RouterView />`，不包含任何界面代码
- **职责**：作为路由视图的容器，由 Vue Router 控制显示哪个页面

### 2. **HomeView.vue** - 门户首页
- **路径**：`/`
- **作用**：公开访问的门户首页，展示平台信息
- **功能**：
  - 展示平台欢迎信息
  - 提供"管理员登录"按钮，跳转到登录页
  - 供普通用户浏览（虽然普通用户主要使用官网和H5端）

### 3. **LoginView.vue** - 登录页面
- **路径**：`/login`
- **作用**：管理员登录页面
- **功能**：
  - 登录表单（学号/邮箱 + 密码）
  - 表单验证
  - 调用登录 API
  - 登录成功后跳转到后台管理首页（`/admin/dashboard`）
  - 已登录用户访问此页面会自动跳转到后台

### 4. **AdminLayout.vue** - 后台管理布局
- **路径**：`/admin/*`（所有后台路由的父布局）
- **作用**：后台管理系统的统一布局组件
- **包含**：
  - 左侧导航栏（侧边栏菜单）
  - 顶部导航栏（面包屑、用户信息、退出登录）
  - 主内容区域（`<router-view />` 显示子路由页面）

### 5. **admin 文件夹下的页面** - 后台管理功能页面
所有后台管理功能页面都在 `src/views/admin/` 目录下，使用 `AdminLayout` 布局。

#### 5.1 DashboardView.vue - 仪表盘
- **路径**：`/admin/dashboard`
- **作用**：数据概览和统计

#### 5.2 UserManagement.vue - 用户管理
- **路径**：`/admin/user`
- **作用**：管理系统用户（查看、编辑、删除用户）

#### 5.3 DepartmentManagement.vue - 部门管理
- **路径**：`/admin/department`
- **作用**：管理部门信息（创建、编辑、删除部门）

#### 5.4 TeamTermManagement.vue - 届次管理
- **路径**：`/admin/team-term`
- **作用**：管理服务队届次信息

#### 5.5 BackboneMemberManagement.vue - 骨干成员管理
- **路径**：`/admin/backbone-member`
- **作用**：管理骨干成员信息

#### 5.6 ActivityManagement.vue - 活动管理
- **路径**：`/admin/activity`
- **作用**：管理志愿活动记录

#### 5.7 ActivityParticipantManagement.vue - 活动参与管理
- **路径**：`/admin/activity-participant`
- **作用**：管理活动参与记录（签到、服务时长等）

#### 5.8 HonorManagement.vue - 荣誉管理
- **路径**：`/admin/honor`
- **作用**：管理荣誉记录

#### 5.9 AnnouncementManagement.vue - 公告管理
- **路径**：`/admin/announcement`
- **作用**：管理公告和通知

#### 5.10 GalleryManagement.vue - 相册管理
- **路径**：`/admin/gallery`
- **作用**：管理团队相册照片

#### 5.11 MilestoneManagement.vue - 发展历程管理
- **路径**：`/admin/milestone`
- **作用**：管理服务队发展历程记录

#### 5.12 OperationLogManagement.vue - 操作日志
- **路径**：`/admin/operation-log`
- **作用**：查看系统操作日志

## 页面流转关系

```
用户访问网站
    ↓
HomeView (/) - 门户首页
    ↓ (点击"管理员登录")
LoginView (/login) - 登录页面
    ↓ (登录成功)
AdminLayout (/admin) - 后台管理布局
    ├── DashboardView (/admin/dashboard) - 仪表盘
    ├── UserManagement (/admin/user) - 用户管理
    ├── DepartmentManagement (/admin/department) - 部门管理
    ├── TeamTermManagement (/admin/team-term) - 届次管理
    ├── BackboneMemberManagement (/admin/backbone-member) - 骨干成员管理
    ├── ActivityManagement (/admin/activity) - 活动管理
    ├── ActivityParticipantManagement (/admin/activity-participant) - 活动参与管理
    ├── HonorManagement (/admin/honor) - 荣誉管理
    ├── AnnouncementManagement (/admin/announcement) - 公告管理
    ├── GalleryManagement (/admin/gallery) - 相册管理
    ├── MilestoneManagement (/admin/milestone) - 发展历程管理
    └── OperationLogManagement (/admin/operation-log) - 操作日志
```

## 路由守卫

- **公开路由**：`/`（首页）、`/login`（登录页）- 无需登录
- **受保护路由**：`/admin/*` - 需要登录才能访问
- **自动跳转**：
  - 未登录用户访问 `/admin/*` → 自动跳转到 `/login`
  - 已登录用户访问 `/login` → 自动跳转到 `/admin/dashboard`

## 页面基础代码

所有管理页面都已包含：
- ✅ 页面标题（使用 `el-page-header`）
- ✅ 数据表格（使用 `el-table`）
- ✅ 加载状态（`v-loading`）
- ✅ 操作按钮（新增、编辑、删除等）
- ✅ 基础样式和布局
- ✅ 预留的 API 调用位置（`loadData` 函数）

后续只需要：
1. 实现具体的 API 调用
2. 添加表单对话框（新增/编辑）
3. 完善业务逻辑
